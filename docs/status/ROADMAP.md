# 로드맵

현재 상태(라이브·병합 기능·미완 항목)는 `docs/status/STATUS.md` 참고.

## 인격체화 3축

프로젝트의 상위 테마는 "AI를 하나의 인격체로 느끼게" 만드는 것이며, 세 축으로 나눈다.

1. **캐릭터/인격** — 고정 페르소나(이름·말투·관계 진화)와 소유자/손님/서버별 어조 차등. **완료(main 병합)**.
2. **자기인지** — 비서가 자신의 구조·데이터·행동 이력을 스스로 조회할 수 있는 능력. **진행 중.**
3. **능동성** — 스스로 판단해 먼저 행동하는 능력(예: 능동적 알림·제안). **예정.**

자기인지·능동성은 캐릭터가 발현되는 구체적 행동으로 취급한다 — 캐릭터 톤을 재사용해 구현한다.

### 자기인지 — 남은 조각

자기인지는 여러 조각으로 나눠 진행한다.

- **조각 C(자기 구조·데이터 조회)** — 스키마 조회·읽기전용 SQL 질의·런타임 정보 제공. **완료(main 병합)**.
- **조각 B(작업 관찰)** — 비서 자신의 행동(도구 호출·응답)을 `actions` 테이블 등에 기록하고, 최근 행동을
  스스로 조회할 수 있게 한다. 설계는 이미 되어 있고 구현 대기.
- **조각 A(사용량 자각)** — 자신의 사용량(시간당 처리한 대화 수 등)을 스스로 조회하는 `status`/
  `my_usage` 성격의 기능. 소유자는 한도 예약 없이 무제한이므로 별도 집계 경로가 필요하다.
- **읽기 소스 확장** — 이후 로그·GitHub 등 조회 가능한 소스를 넓혀간다.

## 단계 로드맵 (2C 이후)

- **2C — 관측**: `actions`/`turns` 로깅과 SDK 훅 기반 에러 로그 수집. 조각 B(작업 관찰)와 맞물린다.
- **2D — 백업**: 기억·데이터의 백업/복구 경로 마련.
- **2E — 능동성**: 인격체화 3축의 세 번째 축. 캐릭터 톤을 재사용해 스스로 판단하고 먼저 행동하는
  기능(예: 능동적 알림).
- **후속 — 웹 UI**: 디스코드 외 채널로 확장.

## 5단계 로드맵 요약

원래의 큰 그림은 5단계였고, 진행하며 순서를 일부 재정의했다(자기인지·멀티유저를 파일셸/브라우저보다
앞당김).

1. **코어/디스코드** — 상주 데몬 + 디스코드 어댑터. **완료.**
2. **파일셸/브라우저** — 원래 계획의 PC 작업 범위. 멀티유저 런타임·자기인지·원격 개발 워크플로우
   (Phase A: 소유자 DM 허용 폴더 내 파일·Bash)로 재정의되어 **진행 중**. 손님용 PC 접근(Phase B)은
   OS 격리 등 선행 과제가 있어 이후로 미룬다.
3. **스케줄러/감지** — 능동성(2E)과 맞물리는 예정 단계.
4. **웹 UI** — 디스코드 이외 채널. 후속(하이브리드·인격체화 완료 이후).
5. **장기 목표/GUI** — 더 먼 미래의 확장 방향.
